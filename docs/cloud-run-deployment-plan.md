# Google Cloud Run Deployment Plan

This document details the step-by-step procedure to deploy the NEES Core Engine — Google Immersion Demo to Google Cloud Run once the suspended Google Cloud billing account is active.

> [!IMPORTANT]
> **Deployment Status**: Planned and Cloud Run-ready, but blocked during Phase 1 because the current Google Cloud billing account is suspended.

---

## 1. Prerequisites & GCP Setup

Ensure you have the Google Cloud SDK (`gcloud`) installed and configured.

1. **Log in to Google Cloud Console**:
   ```bash
   gcloud auth login
   ```
2. **Create/Select Project**:
   ```bash
   gcloud config set project [PROJECT_ID]
   ```
3. **Enable GCP APIs**:
   Enable the required APIs (Artifact Registry, Cloud Run, Secret Manager):
   ```bash
   gcloud services enable artifactregistry.googleapis.com run.googleapis.com secretmanager.googleapis.com
   ```

---

## 2. Docker Configuration

### Backend Dockerfile (`backend/Dockerfile`)
Create the Dockerfile in the `backend/` directory to run the FastAPI app:
```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8080

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
```

## 3. Build & Deploy Steps

Once billing is active, deploy the backend service via Google Artifact Registry and Cloud Run:

### Step 1: Create Artifact Registry Repository
```bash
gcloud artifacts repositories create nees-demo-repo \
    --repository-format=docker \
    --location=us-central1 \
    --description="Docker repository for NEES Core Engine Demo"
```

### Step 2: Configure Secret Manager
Create a secret to store your Gemini API Key safely:
```bash
echo -n "YOUR_GEMINI_API_KEY" | gcloud secrets create gemini-api-key --data-file=-
```

### Step 3: Build and Push Docker Image
Run from the `backend/` folder:
```bash
gcloud builds submit --tag us-central1-docker.pkg.dev/[PROJECT_ID]/nees-demo-repo/backend-image:latest .
```

### Step 4: Deploy to Cloud Run
Deploy the container with environment configurations and secret access:
```bash
gcloud run deploy nees-core-backend \
    --image us-central1-docker.pkg.dev/[PROJECT_ID]/nees-demo-repo/backend-image:latest \
    --region us-central1 \
    --allow-unauthenticated \
    --set-env-vars="GEMMA_MODEL=gemma-4-26b-a4b-it,MOCK_GEMMA_WHEN_MISSING_KEY=false,CORS_ORIGINS=https://YOUR_FRONTEND_URL" \
    --set-secrets="GEMINI_API_KEY=gemini-api-key:latest"
```

## 4. Frontend Deployment

To deploy the Vite + React frontend to Cloud Storage and serve via CDN:

1. **Build Frontend Locally**:
   ```bash
   cd frontend
   npm run build
   ```
2. **Upload Build Folder to Google Cloud Storage**:
   ```bash
   gsutil cp -r dist/* gs://[YOUR_FRONTEND_BUCKET_NAME]/
   ```
3. **Point Frontend to Cloud Run Backend URL**:
   Ensure `VITE_API_BASE_URL` in the frontend production build points to the HTTPS URL generated by the Cloud Run deploy command.
